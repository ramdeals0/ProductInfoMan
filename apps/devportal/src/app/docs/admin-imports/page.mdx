# Import API

Bulk ingest products from **CSV**, **JSON**, or **XML** with validation, error reporting, and commit.

<Callout variant="tip">
  Admin UI: `/admin/imports` and job detail `/admin/imports/[id]`
</Callout>

## Endpoints

<Endpoint method="POST" path="/api/v1/imports/upload" description="Upload a CSV, JSON, or XML file and create an import job." adminPath="/admin/imports" />
<Endpoint method="POST" path="/api/v1/imports/:id/validate" description="Parse and validate rows against templates and rules." />
<Endpoint method="POST" path="/api/v1/imports/:id/start" description="Commit valid rows to the catalog." />
<Endpoint method="GET" path="/api/v1/imports" description="List import jobs." adminPath="/admin/imports" />
<Endpoint method="GET" path="/api/v1/imports/:id" description="Get job status, counters, and file type." />
<Endpoint method="GET" path="/api/v1/imports/:id/rows" description="Preview parsed rows (flattened field paths for JSON/XML)." />
<Endpoint method="GET" path="/api/v1/imports/:id/errors" description="Row-level validation errors." />
<Endpoint method="GET" path="/api/v1/imports/:id/report" description="Summary report after completion." />

## File types

| Type | `file_type` | Notes |
|------|-------------|-------|
| CSV | `csv` | Header row + comma-separated values (default) |
| JSON | `json` | Root array of product objects |
| XML | `xml` | Root element with repeated `<product>` children |

`file_type` is inferred from the file extension when omitted. Override with a query param or multipart field.

## CSV example

```csv
sku,product_type,title,description,brand,category_code,summary,selling_points,start_date,discontinue_date,color,size,fabric,fit,price,material,facet_brand,facet_color,facet_size,facet_fabric,facet_fit,facet_price,facet_material
CSV-001,SIMPLE,Import Oxford Shirt,A classic oxford shirt for everyday wear.,Demo Brand,oxford,Classic oxford shirt with a comfortable everyday fit.,Quality fabric|Easy care|Classic fit,2025-01-01,2027-12-31,Blue,M,Cotton,Regular,49.99,Cotton,Demo Brand,Blue,M,Cotton,Regular,25_to_50,Cotton
```

Use pipe-separated values for `selling_points` in CSV. Attribute columns (`color`, `fabric`, `fit`, `price`, etc.) are imported via your template. `facet_*` columns in downloadable samples show browse facet values derived after indexing and are not mapped by the default template unless you add them explicitly.

## JSON import

Expected shape: a JSON array at the root, or an object with a `products` / `items` array. Nested objects are flattened with dot notation for template mapping (e.g. `attributes.color`).

```json
[
  {
    "sku": "JSON-001",
    "product_type": "SIMPLE",
    "title": "Import Oxford Shirt",
    "description": "A classic oxford shirt for everyday wear.",
    "brand": "Demo Brand",
    "category_code": "oxford",
    "summary": "Classic oxford shirt with a comfortable everyday fit.",
    "selling_points": ["Quality fabric", "Easy care", "Classic fit"],
    "start_date": "2025-01-01",
    "discontinue_date": "2027-12-31",
    "attributes": {
      "color": "Blue",
      "size": "M",
      "fabric": "Cotton",
      "fit": "Regular",
      "price": "49.99",
      "material": "Cotton"
    },
    "facets": {
      "brand": "Demo Brand",
      "color": "Blue",
      "size": "M",
      "fabric": "Cotton",
      "fit": "Regular",
      "price": "25_to_50",
      "material": "Cotton"
    }
  }
]
```

<CodeBlock language="bash" title="Upload JSON">
{`curl -X POST "https://api.yourpim.com/api/v1/imports/upload?file_type=json" \\
  -H "X-Organization-Slug: demo" \\
  -F "file=@products.json" \\
  -F "importType=UPSERT"`}
</CodeBlock>

Map flattened paths in your import template, for example:

- `sku` → `sku`
- `product_type` → `product_type`
- `attributes.color` → `color`

## XML import

Expected shape: a root element (default: first root tag) containing repeated `<product>` elements. Child elements flatten to dotted paths.

Downloadable examples include **SIMPLE**, **PARENT**, and **VARIANT** rows. Variant products reference their parent with `<parent_sku>`.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<products>
  <product>
    <sku>XML-001</sku>
    <product_type>SIMPLE</product_type>
    <title>Import Oxford Shirt</title>
    <description>A classic oxford shirt for everyday wear.</description>
    <brand>Demo Brand</brand>
    <category_code>oxford</category_code>
    <summary>Classic oxford shirt with a comfortable everyday fit.</summary>
    <selling_points>Quality fabric|Easy care|Classic fit</selling_points>
    <start_date>2025-01-01</start_date>
    <discontinue_date>2027-12-31</discontinue_date>
    <attributes>
      <color>Blue</color>
      <size>M</size>
      <fabric>Cotton</fabric>
      <fit>Regular</fit>
      <price>49.99</price>
      <material>Cotton</material>
    </attributes>
    <facets>
      <brand>Demo Brand</brand>
      <color>Blue</color>
      <size>M</size>
      <fabric>Cotton</fabric>
      <fit>Regular</fit>
      <price>25_to_50</price>
      <material>Cotton</material>
    </facets>
  </product>
  <product>
    <sku>XML-003</sku>
    <product_type>PARENT</product_type>
    <title>Import Oxford Shirt Family</title>
    <description>Parent product grouping oxford shirt variants by size and color.</description>
    <brand>Demo Brand</brand>
    <category_code>oxford</category_code>
    <summary>Oxford shirt available in multiple sizes and colors.</summary>
    <selling_points>Multiple fits|Coordinated variants|Easy merchandising</selling_points>
    <start_date>2025-01-01</start_date>
    <discontinue_date>2027-12-31</discontinue_date>
  </product>
  <product>
    <sku>XML-004</sku>
    <product_type>VARIANT</product_type>
    <title>Import Oxford Shirt — Blue / M</title>
    <description>Oxford shirt variant in blue, size medium.</description>
    <brand>Demo Brand</brand>
    <parent_sku>XML-003</parent_sku>
    <category_code>oxford</category_code>
    <summary>Blue oxford shirt in size medium.</summary>
    <selling_points>Quality fabric|Easy care</selling_points>
    <start_date>2025-01-01</start_date>
    <discontinue_date>2027-12-31</discontinue_date>
    <attributes>
      <color>Blue</color>
      <size>M</size>
      <fabric>Cotton</fabric>
      <fit>Regular</fit>
      <price>49.99</price>
      <material>Cotton</material>
    </attributes>
    <facets>
      <brand>Demo Brand</brand>
      <color>Blue</color>
      <size>M</size>
      <fabric>Cotton</fabric>
      <fit>Regular</fit>
      <price>25_to_50</price>
      <material>Cotton</material>
    </facets>
  </product>
</products>
```

Use pipe-separated values for `selling_points` in XML (same as CSV). Nested `<attributes>` elements map to dotted template paths such as `attributes.color`. The optional `<facets>` block in downloadable samples shows computed browse facet values (for example price buckets like `25_to_50`).

<CodeBlock language="bash" title="Upload XML">
{`curl -X POST "https://api.yourpim.com/api/v1/imports/upload?file_type=xml" \\
  -H "X-Organization-Slug: demo" \\
  -F "file=@products.xml" \\
  -F "importType=CREATE"`}
</CodeBlock>

## Full import flow

<CodeBlock language="bash" title="1. Upload file">
{`curl -X POST "https://api.yourpim.com/api/v1/imports/upload" \\
  -H "X-Organization-Slug: demo" \\
  -F "file=@products.json" \\
  -F "file_type=json"`}
</CodeBlock>

<CodeBlock language="bash" title="2. Validate">
{`curl -X POST "https://api.yourpim.com/api/v1/imports/IMPORT_ID/validate" \\
  -H "X-Organization-Slug: demo"`}
</CodeBlock>

<CodeBlock language="bash" title="3. Start commit">
{`curl -X POST "https://api.yourpim.com/api/v1/imports/IMPORT_ID/start" \\
  -H "X-Organization-Slug: demo"`}
</CodeBlock>

<CodeBlock language="bash" title="4. Check errors">
{`curl "https://api.yourpim.com/api/v1/imports/IMPORT_ID/errors" \\
  -H "X-Organization-Slug: demo"`}
</CodeBlock>

## JavaScript upload

<CodeBlock language="typescript">
{`const form = new FormData();
form.append("file", file);
form.append("file_type", "json");
form.append("importType", "UPSERT");

const res = await fetch("https://api.yourpim.com/api/v1/imports/upload", {
  method: "POST",
  headers: { "X-Organization-Slug": "demo" },
  body: form,
});
const job = await res.json();`}
</CodeBlock>

Import templates: `GET /api/v1/import-templates`, `POST /api/v1/import-templates`.

Template `sourceColumn` values work for CSV column names **or** flattened JSON/XML field paths.
