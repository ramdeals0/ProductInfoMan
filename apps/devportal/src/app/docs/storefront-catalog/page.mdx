# Storefront Catalog API

Read-only catalog endpoints used by the headless demo storefront. There is no separate `/storefront/*` prefix — the storefront uses the same `/api/v1` routes with organization context.

<Callout variant="tip">
  These endpoints power:
  - `/search` — site search
  - `/category/[slug]` — category browse with facets
  - `/product/[id]` — product detail
</Callout>

## Authentication

Storefront calls only need:

```
X-Organization-Slug: demo
```

No actor headers required for read-only catalog access.

## Search with facets

<CodeBlock language="bash" title="Amazon-style faceted search">
{`curl "https://api.yourpim.com/api/v1/search?q=drill&categoryId=CATEGORY_ID&filters[brand]=DEWALT&filters[tool_type]=Drill&page=1&pageSize=24" \\
  -H "X-Organization-Slug: demo"`}
</CodeBlock>

Legacy-style query (documented alias — use `filters[key]` in this API):

```bash
# Equivalent facet filter
curl "https://api.yourpim.com/api/v1/search?q=drill&filters[brand]=DEWALT&page=1&pageSize=24" \
  -H "X-Organization-Slug: demo"
```

## Storefront JavaScript client

The demo storefront uses `@productinfoman/api-client` `CatalogClient`:

<CodeBlock language="typescript">
{`import { createCatalogClient } from "@productinfoman/api-client";

const catalog = createCatalogClient({
  baseUrl: "https://api.yourpim.com/api/v1",
  organizationSlug: "demo",
});

const results = await catalog.searchProducts({
  q: "drill",
  categoryId: "CATEGORY_ID",
  filters: { brand: "DEWALT" },
  page: 1,
  pageSize: 24,
});

const product = await catalog.getProduct("PRODUCT_ID");`}
</CodeBlock>

## Checkout

Checkout is handled by the storefront Next.js route `POST /api/checkout` (Stripe or mock). That is **not** part of the PIM API — see the storefront app.
